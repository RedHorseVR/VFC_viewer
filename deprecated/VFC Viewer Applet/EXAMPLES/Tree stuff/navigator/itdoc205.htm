    • Overview of additional features
     
    In the commercial versions you also get:
  • 17 extra parameters
  • index files that can be loaded from parameters as well as external index files
  • the removal of all visible copyright information from the information panel
  • additional index capacity (100 entries in iTree Mini Menu; infinite entries in iTree Express)
  • ability to remove colour-change button in iTree Mini
  • further topbar & tab configuration options in iTree Express
    • 1. Text formatting. To format the text (for the moment excluding colours), there are the following parameters. The parameters must be written into your HTML file between the APPLET tags - just like the examples earlier in the documentation.
     
    <param name=textfont value="Helvetica">
    <param name=textsize value=11>
    <param name=textstyle value=0>
     
    The parameter textfont may have the values "Helvetica" and "TimesRoman". The parameter textsize may have any value you choose. The parameter textstyle follows this scheme: 0 = plain, 1 = bold, 2 = italic, 3 = bold italic.
     
    2. Colour parameters. In the free version you choose from preset colour schemes. In the commercial versions you can set any of 8 colours yourself. To do this, you must first set the colourscheme parameter to 8, which tells you that you are choosing your own colours. Then you must give each of the 8 colour parameters a value. They all take hexadecimal values just like HTML colours. For example:
     
    <param name=bgcol1 value="000000">
    <param name=bgcol2 value="ece4ba">
    <param name=bgcol3 value="6b6331">
    <param name=bgcol4 value="9a7932">
    <param name=bgcol5 value="9a7932">
    <param name=textcol1 value="ba9952">
    <param name=textcol2 value="ece4ba">
    <param name=textcol3 value="6b6331">
     
    The colour effects are as follows:
  • bgcol1 = box top-left border
  • bgcol2 = box bottom-right border
  • bgcol3 = box main frame fill
  • bgcol4 = menu background
  • bgcol5 = scroll background
  • textcol1 = menu text colour
  • textcol2 = title text colour
  • textcol3 = rectangular background behind text on mouse-over
    • 3. Customising the tab and title texts
     
    Enter the following parameter with 2 pieces of text separated by a vertical line |.
     
    <param name=texts value="Custom Title|Put your own text for the information panel here. § § Enjoy it! § The Author">
     
    The first piece of text alters the applet title bar text. The final piece of text changes the text that appears in the information panel. In this final piece of text, space-§-space may be used to force line breaks. In iTree Express, this parameter has additional functions - see the additional documentation for iTree Express.
     
    4. Loading an index from parameters
     
    The advantage of loading an index from parameters is that the applet starts faster (but your page loads slower). The pros and cons of external indices and parameter indices are complex, and some people prefer one or the other. With the commercial versions of this applet, you have the choice.
     
    The parameter index loading uses a special parameter called "entry". This parameter can be used up to about 100 times, each time with an integer tagged on to its name. So you can start at "entry0" for the first entry, "entry1" for the second entry, and so on.
     
    <PARAM NAME="entry0" VALUE="....">
    <PARAM NAME="entry1" VALUE="....">
    <PARAM NAME="entry2" VALUE="....">
    <PARAM NAME="entry3" VALUE="....">
    <PARAM NAME="entry4" VALUE="....">
    <PARAM NAME="entry5" VALUE="....">
     
    The format of the entry parameter's value is much the same as the format of a single line in the index file (described earlier), with one difference: instead of separating the individual elements with inverted commas, they must be separated with the vertical line symbol |. Thus:
    "TEXT:my entry" "1" "" "LINK:myPage.html,myFrame" "description of my page"
    becomes:
    <PARAM NAME="entry_" VALUE="TEXT:my_entry|1||LINK:myPage.html,myFrame|description of my page">
     
     5. Customising the title-bar buttons
     
    Enter the following parameter and then adjust the value for the desired effect.
     
    <param name=buttons value=0>
     
  • Value 0: without colour-adjustment button
  • Value 1: with colour-adjustment button
     
    This parameter has extra functions in iTree Express - see the additional documentation for iTree Express.
     
    6. Further parameters
     
    Menufile parameter: you can change the index filename ("menu.txt" in the free version) to whatever you like; script-writers may find this has advantages, as you can load menu content dynamically from such things as CGI scripts.
     
    Startlevel parameter: you can enter an integer here to determine sub-menu levels that are open when the applet first starts. Experiment with values such as 0, 1, 2 and 3.
     
    Scrollwidth parameter. The scrollbar can be given a width between 7 and 19 pixels.
